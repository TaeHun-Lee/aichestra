# Production Deployment Readiness Planning v0

## Status

`v0_implemented` as planning/readiness metadata only.

Aichestra is not production-ready. This milestone adds a production readiness architecture, inventories, checklists, risk register, deployment profile models, read-only API endpoints, and dashboard visibility. It does not deploy anything and does not enable production traffic.

## Deployment Profiles

- `local`: mock-first, in-memory default, no external calls.
- `integration`: optional Postgres and gated GitHub/OpenAI-compatible paths, still no production traffic.
- `staging`: planned profile requiring Postgres, migrations, real auth, real secret backend, audit retention, and no auto-merge.
- `production`: blocked target profile requiring production auth, real secret backend, policy bundles, observability, audit retention, backup/restore, tenancy, quotas, and mock actor rejection.

## Production Blockers Summary

- Production Auth/RBAC is not implemented.
- Real secret backend is not implemented.
- Postgres is opt-in and repository coverage is incomplete for all production stores.
- Policy bundle management is not implemented.
- Observability and audit export are not implemented.
- Backup/restore operations are not implemented.
- Tenant isolation is not implemented.
- Legacy env credential fallback remains for compatibility outside production.

## Safety

Readiness endpoints are read-only and local/seeded. They perform no external checks and expose no secrets. They do not call GitHub, GitLab, Bitbucket, LLM providers, MCP servers, Vault, Kubernetes, Temporal, artifact registries, vendor CLIs, or production auth systems.

## Known Limitations

- Readiness checks are static seeded planning data, not live infrastructure probes.
- No production deployment profile enforcement exists yet.
- No infra manifests or cloud resources exist.
- No production auth, secret backend, policy bundle runtime, observability backend, or tenant boundary is implemented.
