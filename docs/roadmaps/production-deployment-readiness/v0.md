# Production Deployment Readiness Planning v0

## Status

`v0_implemented` as planning/readiness metadata only.

Aichestra is not production-ready. This milestone adds a production readiness architecture, inventories, checklists, risk register, deployment profile models, read-only API endpoints, and dashboard visibility. It does not deploy anything and does not enable production traffic.

Observability / Audit Retention v0, External Observability Export v1, GitHub App / Production Webhook Hardening Planning v0, GitHub App Controlled Implementation v1, Persistent DB Production Operations v1, Durable Collaboration Stores v1, Secret Backend Migration Planning v0, Vault-backed Secret Backend v1, Production Auth/RBAC v1 Planning, Policy Bundle / OPA-Cedar Planning v0, and Staging Deployment Profile v0 are now implemented as read-only foundations or controlled mock/status boundaries. Production observability, production webhooks, production database operations, production secret management, production auth, production policy bundle runtime, production collaboration-store enforcement, and real staging deployment remain blocked because there is no durable observability backend, real audit export pipeline, alert delivery, retention deletion/legal-hold workflow, real GitHub App private-key signing/token exchange, durable replay store, queue/dead-letter worker, hardened production webhook endpoint, production DB pool, backup/restore job, migration governance enforcement, tenant-aware production DB profile, service-by-service durable collaboration source-of-truth migration, production Vault rollout, production rotation workflow, production credential issuance, real IdP adapter, session model, SCIM sync, service-account credential issuance, tenant isolation enforcement, staging CI/CD pipeline, or deployable staging infrastructure.

## Deployment Profiles

- `local`: mock-first, in-memory default, no external calls.
- `integration`: optional Postgres and gated GitHub/OpenAI-compatible paths, still no production traffic.
- `staging`: first-class v0 non-production readiness profile with explicit integration gates, required/forbidden env gates, promotion criteria, rollback criteria, and dashboard/API visibility. It is not deployed and is not production-ready.
- `production`: blocked target profile requiring production auth, real secret backend, policy bundles, observability, audit retention, backup/restore, tenancy, quotas, and mock actor rejection.

## Production Blockers Summary

- Production Auth/RBAC v1 Planning is implemented as read-only readiness metadata, but real OIDC/SAML/SCIM, SSO, sessions, password auth, JWT issuance, service-account credential issuance, and tenant isolation enforcement are not implemented.
- Secret Backend Migration Planning v0 documents backend options, SecretRef migration, credential kind migration, lease/rotation, env fallback deprecation, readiness APIs, health metadata, and dashboard visibility. Vault-backed Secret Backend v1 implements the gated `vault` provider boundary, but production Vault rollout and migration remain blocked.
- Postgres is opt-in and repository coverage is incomplete for all production stores.
- Persistent DB Production Operations v1 documents and exposes readiness metadata, but production pooling, backup/restore jobs, migration governance enforcement, retention/legal hold, partitioning, and tenant scoping remain unimplemented.
- Durable Collaboration Stores v1 adds safe in-memory default and optional Postgres repository/schema boundaries for collaboration records, but production DB operations, distributed locks, tenant-aware enforcement, and service-by-service durable source-of-truth migration remain unimplemented.
- Policy bundle management is not implemented.
- Observability v0 read models and External Observability Export v1 skeletons are implemented, but production observability backend, real audit export, alerting, and retention enforcement are not implemented.
- GitHub App / Production Webhook Hardening Planning v0 and GitHub App Controlled Implementation v1 are implemented, but live GitHub App private-key signing, real installation token exchange, production webhook endpoint rollout, durable replay/idempotency storage, and retry/dead-letter worker are not implemented.
- Backup/restore operations are not implemented.
- Tenant isolation is not implemented.
- Legacy env credential fallback remains for compatibility outside production.

## Safety

Readiness endpoints are read-only and local/seeded. They perform no external checks and expose no secrets. They do not call GitHub, GitLab, Bitbucket, LLM providers, MCP servers, Vault in default runtime/tests, Kubernetes, Temporal, artifact registries, vendor CLIs, or production auth systems.

## Known Limitations

- Readiness checks are static seeded planning data, not live infrastructure probes.
- No production deployment profile enforcement exists yet.
- No infra manifests or cloud resources exist.
- No production auth implementation, secret backend, policy bundle runtime, signed bundle verification, durable observability backend, audit export, retention enforcement, or tenant boundary is implemented. Policy Bundle / OPA-Cedar Planning v0 is read-only readiness metadata only.
