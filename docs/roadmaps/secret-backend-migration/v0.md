# Secret Backend Migration Planning v0

Status: v0_implemented.

This is a planning/readiness milestone. It does not implement real Vault, AWS Secrets Manager, GCP Secret Manager, Azure Key Vault, custom secret backend, BYOK, OAuth, WIF/IAM, cloud identity token exchange, production rotation jobs, credential issuance, or actual secret migration.

## What v0 implements

- Backend option comparison and production suitability model.
- SecretRef provider migration strategy from env fallback toward real backends.
- Credential kind migration plan for GitHub, webhook, LLM, provider API, MCP, Local Agent, OAuth/cloud identity, service-account signing, and future BYOK credentials.
- Lease TTL and rotation planning models.
- Env fallback deprecation plan by deployment profile.
- Read-only `SecretBackendMigrationReadinessService` behavior inside `DeploymentReadinessService`.
- Read-only DTO endpoints:
  - `GET /readiness/secrets/backends`
  - `GET /readiness/secrets/migration-phases`
  - `GET /readiness/secrets/checks`
  - `GET /readiness/secrets/risks`
  - `GET /readiness/secrets/rotation-plans`
  - `GET /readiness/secrets/lease-policies`
  - `GET /readiness/secrets/summary`
- `/health` secret backend migration metadata with booleans/counts/status only.
- Dashboard Secret Backend Migration panel.
- Deterministic tests for models, APIs, dashboard, health metadata, production env fallback blocking, and no-secret/no-env exposure.

## What v0 does not implement

- No real secret backend connection.
- No real secret reads, writes, export, or migration.
- No production secret rotation jobs.
- No credential issuance.
- No BYOK, OAuth, device-code, WIF, IAM, or cloud identity exchange.
- No vendor CLI execution.
- No credential cache reads.
- No secret injection into runner, MCP, Local Agent, Git, LLM, or provider processes.

## Target architecture

`SecretRef` remains the durable application-facing reference. Future backends supply value storage and backend-native audit/rotation/version controls behind the existing security package contracts. API handlers, workers, dashboard read models, and provider gateways continue to see metadata, handles, and status only.

Production profile requirements:

- selected real backend;
- production Auth/RBAC and service accounts;
- Policy-as-code approval for credential purposes;
- metadata-only audit and observability correlation;
- env fallback disabled;
- no provider credential cache access;
- rotation/revocation runbooks.

## Backend options

See `backend-options-v0.md`.

Summary:

- Vault is recommended when already operated or dynamic leases are required.
- AWS Secrets Manager, GCP Secret Manager, and Azure Key Vault are strong cloud-native options.
- Custom enterprise backends remain future adapters.
- Env fallback is local/integration only and not production suitable.

## SecretRef provider migration

See `secretref-provider-migration-v0.md`.

Migration keeps stable SecretRef ids while backend provider metadata changes. Secret versions are represented as metadata only. Disabled/revoked refs fail closed. Production rollback disables provider gates rather than re-enabling env fallback.

## Credential kind migration

See `credential-kind-migration-v0.md`.

The first planned credential kinds are:

- `github_token`
- `github_webhook_secret`
- `github_app_private_key`
- `llm_api_key`

Future-only kinds include provider API keys, OAuth tokens, cloud identity, MCP tool secrets, Local Agent pairing secrets, service account signing keys, and BYOK.

GitHub App Controlled Implementation v1 adds the `github_app_private_key` SecretRef kind as metadata-only runtime support for mock token-handle checks. This does not change the migration blocker: live private-key signing and installation-token exchange still require a real secret backend and reviewed operational controls.

## Lease TTL and rotation

See `lease-ttl-rotation-v0.md`.

v0 models TTL and rotation expectations only. It does not enforce backend TTLs or run scheduled rotation.

## Env fallback deprecation

See `env-fallback-deprecation-v0.md`.

Local may allow env fallback explicitly. Integration may allow it with warnings. Staging and production should block it after real backend migration.

## Auth/RBAC integration

Production Auth/RBAC remains planning/mock-first. Production Auth/RBAC v1 Planning now documents provider options, tenant/scope requirements, service-account actors, request context propagation, mock actor deprecation, and the production permission matrix. It still does not implement real IdP calls, sessions, JWTs, service-account credential issuance, or tenant enforcement. Secret Backend Migration v0 documents that:

- security admins approve backend migration and rotation plans;
- service accounts resolve backend-backed credentials for narrow purposes;
- ordinary users cannot view raw secrets;
- credential audit metadata is visible only through sanitized read models;
- break-glass remains a future explicit design.

## Policy integration

Policy-as-code remains static TypeScript. Secret backend migration must keep deny-by-default secret read behavior. Future production profile policy must deny env fallback and deny disabled/revoked SecretRefs.

## Observability integration

Planned metrics:

- credential resolution attempts;
- credential resolution denials;
- env fallback usage count;
- SecretRef revoked/disabled usage attempts;
- rotation overdue count;
- lease issue/revoke counts;
- secret backend errors;
- no-secret exposure violations.

Planned audit events:

- `secret_backend_selected_future`
- `secret_ref_migrated_future`
- `secret_rotation_requested_future`
- `secret_rotation_completed_future`
- `secret_backend_unavailable_future`
- `env_fallback_used_warning`
- `env_fallback_denied_in_production`

No external audit export is implemented in v0.

## Dashboard/API changes

Dashboard shows:

- backend options;
- env fallback warning;
- credential kind migration status;
- lease TTL and rotation readiness;
- production blockers;
- risks;
- no-secret/no-env status.

API/readiness routes are read-only and deterministic.

## Known limitations

- No production secret backend is selected or integrated.
- No production Auth/RBAC implementation, service account token issuance, policy bundle runtime, tenant boundary enforcement, or backend identity exchange exists.
- No scheduled rotation, stale credential detector, or backend outage handling exists.
- Env fallback deprecation is modeled and tested but not production-enforced by a live profile gate.
