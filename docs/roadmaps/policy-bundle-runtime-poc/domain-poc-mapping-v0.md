# Policy Runtime PoC Domain Mapping v0

Status: planning/readiness only
Runtime impact: none

`StaticPolicyEngine` remains the runtime source of truth. The mappings below define future PoC fixtures and bundle representations only. Policy Runtime PoC Golden Test Harness v1 now realizes a safe subset of these mappings as static TypeScript fixtures and StaticPolicyEngine comparisons; it does not implement runtime bundle execution.

| PoC domain | Current static action | Current resource kind | Required subject fields | Required scope fields | Required environment gates | Expected deny-by-default behavior | PoC test cases | Future bundle representation |
|---|---|---|---|---|---|---|---|---|
| Git remote operation | `git.remote_operation`, `git.merge`, `git.rebase`, branch delete/force-push class | `git_operation` | actorId, roles, authMode | tenantId, projectId, repoId | remote Git gate, operation gate, repo allowlist, branch prefix | merge, rebase, force push, branch deletion, unknown destructive Git deny | `git_remote_merge_denied`, `git_force_push_denied` | `git-remote-operations` |
| GitHub App token issuance | `github_app.installation_token.issue` | `github_app_installation` | actorId, serviceAccountId, roles, authMode | tenantId, repoId, providerId | GitHub App mode, enable gate, installation allowlist, repo allowlist, private-key SecretRef metadata | real token issuance is not implemented; token handles are metadata-only | `service_account_cannot_bypass_policy` | `github-app-token-handle` |
| GitHub webhook processing | `git.webhook.process` | `git_operation` | actorId, serviceAccountId, authMode | tenantId, repoId | webhooks enabled, signature verified, repo allowlist, replay check | unverified or unallowlisted webhooks deny; raw payload is not stored | `github_webhook_unverified_denied` | `git-webhook-receiver` |
| LLM remote completion | `llm.remote_completion` | `llm_provider` | actorId, principalId, roles, authMode | tenantId, providerId, modelId | remote LLM, completion gate, base URL configured, credential metadata, model allowlist, budget allow | remote completion denies unless every gate and policy check passes | `llm_remote_completion_requires_gates` | `llm-remote-completion` |
| LLM fallback | `llm.fallback` | `llm_fallback_policy` | actorId, roles, authMode | tenantId, providerId, modelId | fallback enabled, bounded attempts, provider/model allowlist, budget allow | fallback disabled by default; unbounded fallback denies | `llm_fallback_bounded` | `llm-fallback` |
| MCP tool invocation | `mcp.tool.invoke`, `mcp.tool.invoke.critical` | `mcp_tool` | actorId, roles, authMode | tenantId, mcpServerId, mcpToolId | mock gateway active, low risk, real transport disabled, secret forwarding disabled, network disabled | critical/high-risk/write/deploy/network/secret tools deny | `mcp_critical_tool_denied` | `mcp-tool-risk` |
| SecretRef/Vault credential resolution | `provider.credential.resolve`, `secret.read`, `secret.lease.request` | `secret_ref` | actorId, serviceAccountId, roles, authMode | tenantId, providerId, secretRefId | active SecretRef, Auth/RBAC allow, Policy allow, Vault gates, path allowlist | raw secret read and credential-cache read deny; Vault resolution requires all gates | `secret_read_denied`, `credential_cache_read_denied`, `vault_secret_resolution_requires_auth_policy_path_allowlist` | `secretref-resolution` |
| Runner command execution | `runner.command.execute`, `runner.secret.inject` | `command` | actorId, roles, authMode | tenantId, projectId, repoId | local execution gate, harness allow, fixture workspace, network disabled, secret injection denied | command execution and secret injection deny by default | `runner_secret_injection_denied` | `runner-execution` |
| Local Agent invocation | `local_agent.invoke`, `local_agent.secret.forward`, `credential.cache.read` | `local_agent` | actorId, principalId, roles, authMode | tenantId, localAgentHostId | mock transport, connected host, consent, sandbox, redaction | real transport, PTY, secret forwarding, cache reads, vendor CLI execution deny | `credential_cache_read_denied`, `service_account_cannot_bypass_policy` | `local-agent-consent` |
| Registry mutation | `registry.create`, `registry.update`, `registry.approve`, `registry.rollback` | `registry_item` | actorId, principalId, roles, authMode | tenantId, registryPackageId | authorizer allow, lifecycle, approval, checksum, eval gates | pending approval entries excluded; mutation roles required | `registry_pending_approval_excluded` | `registry-governance` |
| Governance apply gate | `improvement.apply` | `improvement_proposal` | actorId, serviceAccountId, roles, authMode | tenantId, registryPackageId | human approval, eval passed, canary ready, auto apply disabled | governance apply denied and auto-apply disabled | `governance_apply_gate_denied` | `improvement-governance` |
| Dashboard/readiness access | `dashboard.read`, `readiness.read` | `dashboard` | actorId, principalId, roles, authMode | tenantId, teamId, projectId, audit query scope | read-only route, sanitized DTO, no secret/env values | viewers see safe summaries only; tenant mismatch must deny in future | `dashboard_viewer_no_secret_adjacent_details`, `tenant_scope_mismatch_denied_future` | `dashboard-read-scope` |
| Observability audit query | `observability.audit.read`, `policy.audit.read` | `dashboard` | actorId, principalId, roles, authMode | tenantId, audit query scope, retention class | sanitized audit envelope, retention class allowed, raw sensitive views denied | security_admin sees sanitized audit metadata only | `security_admin_audit_metadata_no_raw_secrets` | `audit-query-scope` |

All future bundle representations must preserve deny-by-default behavior and must not bypass Auth/RBAC, Policy, SecretRef, Git, LLM, MCP, Local Agent, Runner, Dashboard, Observability, Registry, Governance, Staging, CI/CD, or Secrets/Sandbox controls.

## Golden Harness v1 Coverage

The v1 harness adds 42 static cases across Git, GitHub App token-handle issuance, LLM, MCP, SecretRef/Vault, Runner, Local Agent, Registry/Governance, Dashboard/readiness, and tenant/scope metadata. A few future domains are represented through current static actions such as `dashboard.read` and `audit.secret.view` until dedicated readiness/audit policy actions exist.

## Shadow Evaluation Planning v1 Coverage

Policy Runtime Shadow Evaluation Planning v1 uses the same domain list for future static-vs-candidate comparison reporting. Each future candidate runtime must declare supported domains, compare against StaticPolicyEngine decisions, and classify mismatches without changing enforcement. No candidate runtime or shadow evaluator is implemented.
