## What v1 Does Not Implement

v1 does not implement real OIDC, SAML, SCIM, SSO, login/logout, password auth, production sessions, JWT issuance, token validation, session cookie parsing, external identity provider calls, real service-account credential issuance, tenant isolation, or durable auth repositories.

It does not expose secrets, tokens, cookies, session ids, IdP client secrets, raw identity assertions, or provider credentials.

## Target Production Architecture

The target architecture keeps Auth/RBAC behind provider-neutral boundaries:

```text
HTTP/API/Webhook/Worker request
  -> production AuthProvider future
  -> RequestContext/AuthContext
  -> tenant/workspace/project/repo scoped authorization
  -> Policy-as-code decision
  -> SecretRef/credential/sandbox/provider gates
  -> sanitized audit and dashboard/readiness metadata
```

Policy denial remains authoritative for every role, including admins and future service accounts.

## Service Account Plan

See `docs/roadmaps/auth-rbac-production/service-account-system-actor-v1.md`.

Worker, Git webhook, Git provider, LLM Gateway, MCP Gateway, Local Agent Protocol, deployment, and observability export service accounts are modeled as scoped principals. Service Account Actor Boundary v1 implements a static mock catalog and context factory for selected runtime attribution, but no credentials are issued in v1 planning or runtime.

GitHub App Controlled Implementation v1 adds runtime permission catalog entries for `github_app.configure`, `github_app.installation.use`, `github_app.repo_grant.use`, `github_app.installation_token.issue`, and `git.changed_files.read`. These permissions authorize only metadata and mock token-handle boundaries in the current runtime; they do not issue real service-account credentials, bypass Policy-as-code, or allow private-key/token exposure.

Service Account Actor Boundary v1 adds active mock entries such as `git_webhook_service`, `git_sync_service`, `github_app_token_service`, `git_provider_service`, `llm_gateway_service`, `llm_router_service`, `mcp_gateway_service`, `secretref_credential_service`, `vault_credential_resolver_service`, `runner_service`, `local_agent_protocol_service`, `registry_governance_service`, and `improvement_governance_service`. Session, token, cookie, JWT, API-key, and service-account credential boundaries are defined in `docs/roadmaps/auth-rbac-production/session-token-boundary-v1.md` and surfaced as future-only read models by Production Auth Provider Skeleton v1. They remain design only and have `tokenIssued: false` and `validationEnabled: false`.

## Mock Actor Deprecation

See `docs/roadmaps/auth-rbac-production/mock-actor-deprecation-v1.md`.

Mock actor behavior remains allowed for local/tests. Production profile must reject mock actors in a future implementation after real AuthProvider, tenant scoping, and break-glass planning are complete.

## Policy Integration

Production RBAC does not replace Policy-as-code. Policy continues to deny remote Git, unverified webhooks, merge/rebase/force-push/delete, remote LLM completion without gates, unsafe MCP paths, runner command execution, secret reads, credential cache reads, Local Agent secret forwarding, and improvement apply.

Policy Bundle / OPA-Cedar Planning v0 maps future production Auth/RBAC subjects, service accounts, and tenant scopes into policy bundle inputs, but it remains read-only planning. It does not enable OPA/Rego, Cedar, external policy services, dynamic policy execution, signed bundle verification, rollout, rollback, or break-glass execution.

## SecretRef Integration

SecretRef-backed credential resolution continues to require Auth/RBAC and Policy checks before any env SecretRef lookup. Production credential resolution requires real secret backend migration and scoped service-account identity.

## Dashboard Panel

The Auth/RBAC Production Readiness panel shows provider options, production auth disabled status, mock actor warning, service account readiness, tenant/scope readiness, request context warning, role matrix summary, blockers, and no-token/no-session status.

It never shows tokens, cookies, session ids, passwords, raw IdP assertions, provider credentials, or env values.

The Production Auth Provider Skeleton panel shows active provider `mock`, selected future provider status, production auth disabled, token validation disabled, future session boundary plans, identity mapping readiness, missing config names/counts, blockers, and no-token/no-session/no-env status. It must not imply production auth is active.

## Known Limitations

- No real auth provider.
- No login/session/JWT/password flow.
- No SCIM sync or deprovisioning.
- No tenant-scoped repositories or dashboard filters.
- No production-grade tenant scope enforcement; Tenant Scope Enforcement v1 is partial representative metadata only.
- No production service-account credential issuance.
- No break-glass workflow.
- Auth/RBAC readiness models are deterministic seed/read models, not live IdP checks.
