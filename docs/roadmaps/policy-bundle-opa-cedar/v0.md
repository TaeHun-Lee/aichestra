# Policy Bundle / OPA-Cedar Planning v0

Status: v0_implemented
Production-ready: no
Runtime policy engine: `StaticPolicyEngine`

## What v0 Implements

- Read-only policy engine option models.
- Read-only policy bundle plan models.
- Policy domain mapping for existing static policy domains.
- Policy bundle readiness checks, risks, migration phases, and summary.
- Policy Runtime Shadow Evaluation Planning v1 now defines the future static-vs-candidate comparison architecture, and Shadow Evaluator Skeleton v1 adds disabled/mock evaluator metadata without implementing a live shadow evaluator.
- `/readiness/policy-bundles/*` API endpoints.
- `/dashboard/policy-bundles` panel and dashboard overview metrics.
- `/health` policy bundle readiness metadata.
- Documentation for engine comparison, bundle schema, review workflow, testing, rollout/rollback, break-glass, and domain mapping.
- Cross-reference to Policy Bundle Runtime PoC Planning v0 for future runtime proof-of-concept evaluation without runtime implementation.
- Cross-reference to Policy Runtime PoC Golden Test Harness v1 for offline StaticPolicyEngine golden comparisons.
- Cross-reference to Policy Runtime Shadow Evaluation Planning v1 and Shadow Evaluator Skeleton v1 for future candidate runtime comparison planning without live shadow evaluator execution.

## What v0 Does Not Implement

- OPA/Rego runtime.
- Cedar runtime.
- Policy Bundle Runtime PoC evaluator.
- External policy decision service.
- Dynamic policy execution.
- `eval`, `Function`, WASM policy execution, or user-provided policy code.
- Remote policy bundle loading.
- Production policy hot reload.
- Signed bundle verification.
- Policy rollout, rollback, promotion, or break-glass execution.

## Target Architecture

The target path is staged:

1. Keep `StaticPolicyEngine` as runtime.
2. Define bundle schemas and domain mappings.
3. Add non-executing bundle dry-run and validation.
4. Add review, approval, tests, and future signing.
5. Add shadow evaluation planning and then future shadow evaluation against static decisions.
6. Add controlled per-domain runtime activation only after production Auth/RBAC, tenant scoping, audit, and rollback controls exist.

GitHub App Controlled Implementation v1 remains on the current `StaticPolicyEngine`. It adds static rules for GitHub App configure, installation use, repo-grant use, and mock installation token-handle issuance. These rules deny by default unless GitHub App mode, enable gates, app id, active private-key SecretRef metadata, installation allowlist, repo grant allowlist, Auth/RBAC allow, and non-destructive operation context all pass.

## Engine Comparison

The near-term recommendation is a schema-driven signed JSON/YAML policy bundle bridge. OPA/Rego remains a strong future candidate for cross-domain operational policy. Cedar remains a future candidate for authorization-heavy domains. Static TypeScript remains acceptable for the current mock/gated milestone but not for production governance.

## Runtime PoC Planning Relationship

`docs/roadmaps/policy-bundle-runtime-poc/v0.md` now defines the future runtime PoC evaluation package: goals, candidate runtime options, normalized policy input/output contract, PoC domain mapping, shadow evaluation plan, golden decision cases, and read-only readiness/API/dashboard status.

Policy Runtime PoC Golden Test Harness v1 now implements offline deterministic fixtures and StaticPolicyEngine comparison for the future runtime path. Policy Runtime Shadow Evaluation Planning v1 defines future candidate runtime interface expectations, comparison rules, mismatch taxonomy, reporting, and rollout/rollback metadata. Shadow Evaluator Skeleton v1 adds disabled/mock comparison helpers and fixture reports. `StaticPolicyEngine` is still the active runtime and no PoC evaluator, live shadow evaluator, OPA/Rego, Cedar, signed JSON/YAML runtime, custom policy service, remote bundle loader, or hot reload behavior exists.

## Known Limitations

- Static TypeScript remains the only runtime.
- Bundle schema is documentation only.
- No production Auth/RBAC tenant enforcement exists.
- No policy bundle signing or verification exists.
- Shadow evaluation planning exists in `docs/roadmaps/policy-bundle-runtime-poc/shadow-evaluation-v1.md`, and the disabled/mock skeleton exists in `docs/roadmaps/policy-bundle-runtime-poc/shadow-evaluator-skeleton-v1.md`; no live shadow evaluator or candidate runtime execution exists.
- No production rollout or rollback exists.
- No runtime PoC evaluator or live shadow evaluator exists.
- Golden Harness v1 is static/test-only and is not a policy bundle runtime.
- Shadow Evaluation Planning v1 is readiness metadata only and does not mean shadow comparison is running.

## Recommended Next Task
