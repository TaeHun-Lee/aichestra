# GitHub App Integration-Test Profile v1

Status: v1_implemented

GitHub App integration-test profile v1 defines a controlled optional profile for validating GitHub App branch, PR, changed-file, and webhook fixture behavior. It is read-only and safe by default. Default tests skip all live GitHub behavior.

## What v1 Implements

- `GitHubAppIntegrationTestProfile`
- `GitHubAppIntegrationTestCase`
- `GitHubAppIntegrationTestSafetyCheck`
- `GitHubAppIntegrationTestReadinessSummary`
- Readiness service methods:
  - `getGitHubAppIntegrationTestProfile()`
  - `listGitHubAppIntegrationTestCases()`
  - `listGitHubAppIntegrationTestSafetyChecks()`
  - `getGitHubAppIntegrationTestReadinessSummary()`
  - `canRunGitHubAppIntegrationLiveTests()`
- Read-only APIs:
  - `GET /readiness/github-app-integration/profile`
  - `GET /readiness/github-app-integration/test-cases`
  - `GET /readiness/github-app-integration/safety-checks`
  - `GET /readiness/github-app-integration/summary`
- Health metadata under `githubAppIntegrationTests`.
- Dashboard read model and panel under `/dashboard/github-app-integration`.
- A skipped optional live-test skeleton.

## What v1 Does Not Implement

- Production GitHub App rollout.
- Real GitHub App private-key signing in default runtime/tests.
- Live installation-token exchange in default runtime/tests.
- Production webhook endpoint.
- Auto-merge, rebase, force-push, branch deletion, remote cleanup, or destructive operations.
- GitLab or Bitbucket.

## Required Gates

Live tests can run only when every required gate is configured and no unsafe gate is present:

| Gate | Required value |
|---|---|
| `AICHESTRA_GITHUB_APP_INTEGRATION_TESTS` | `true` |
| `AICHESTRA_ENABLE_REMOTE_GIT` | `true` |
| `AICHESTRA_GITHUB_AUTH_MODE` | `github_app` |
| `AICHESTRA_ENABLE_GITHUB_APP` | `true` |
| `AICHESTRA_GITHUB_APP_ID` | non-empty |
| `AICHESTRA_GITHUB_APP_PRIVATE_KEY_SECRET_REF` | non-empty |
| `AICHESTRA_GITHUB_APP_ALLOWED_INSTALLATIONS` | non-empty |
| `AICHESTRA_GITHUB_APP_ALLOWED_REPOS` | non-empty |
| `AICHESTRA_GITHUB_APP_ALLOWED_BRANCH_PREFIX` | `ai/` |
| `AICHESTRA_ALLOW_REMOTE_BRANCH_CREATE` | `true` |
| `AICHESTRA_ALLOW_REMOTE_PR_CREATE` | `true` |
| `AICHESTRA_ALLOW_REMOTE_MERGE` | `false` |

Webhook tests additionally require `AICHESTRA_GITHUB_WEBHOOK_SECRET_REF`, `AICHESTRA_ENABLE_GITHUB_WEBHOOKS=true`, and `AICHESTRA_GITHUB_WEBHOOK_ACCEPT_UNVERIFIED=false`.

## Skip Behavior

Missing gates skip optional live tests. They do not fail the default test suite. Unsafe gates are reported as unsafe and make `canRunLiveTests=false`.

Unsafe gates include:

- `AICHESTRA_ALLOW_REMOTE_MERGE=true`
- future force-push or branch-delete gates enabled
- `AICHESTRA_GITHUB_WEBHOOK_ACCEPT_UNVERIFIED=true`
- a branch prefix other than `ai/`
- raw GitHub App private key env values

## Test Repository Requirements

Only explicitly allowlisted non-production repositories may be used. Readiness APIs and dashboard return allowlist counts and redacted placeholders, not env values.

## Test Branch and PR Naming

All future live branch and PR tests must use the `ai/` prefix and a clearly test-only title. PR creation must not imply merge permission.

## Cleanup Policy

Cleanup remains manual close/mark-only. Branch deletion is forbidden in v1, including for cleanup.

## Webhook Tests

Default tests may use signed webhook fixtures. Live webhook delivery is future-only because there is no production webhook endpoint or public staging endpoint in this profile.

## SecretRef and Redaction

Private key and webhook secret material must be represented through SecretRef metadata. APIs, health, dashboard, and audit data must not expose private keys, installation tokens, webhook secrets, legacy GitHub tokens, or env values.

## API and Dashboard

The readiness API is read-only and does not call GitHub. The dashboard panel shows profile status, live-test skip status, missing gate count, unsafe gate count, allowed repo count, branch prefix state, test cases, safety checks, cleanup policy, and no-secret status. Staging Deployment Dry-run Profile v0 consumes this profile as an optional source and classifies it as skipped/gated/ready/blocked without executing live tests. Staging Release Candidate Checklist v0 consumes the skip classification and requires release notes to document the optional skip before RC designation. Staging Deployment Execution Plan v0 consumes the same optional decision as a pre-deployment gate without running GitHub tests.

## Known Limitations

- The live test skeleton is intentionally skipped even if gates are present because v1 does not implement a live GitHub App provider execution path.
- Cleanup automation is not implemented.
- Webhook live delivery is future-only.
- Production rollout remains blocked by production auth, real secret backend, DB operations, and production webhook hardening work.
