# LLM Gateway Integration-Test Profile v1

Status: `v1_implemented`

LLM Gateway integration-test profile v1 defines a controlled, skipped-by-default profile for validating the OpenAI-compatible / LiteLLM-compatible remote LLM boundary in non-production environments. It does not enable live LLM calls by default and does not make LLM integration production-ready.

## What v1 Implements

- `LLMIntegrationTestProfile`, `LLMIntegrationTestCase`, `LLMIntegrationTestSafetyCheck`, and `LLMIntegrationTestReadinessSummary` read models in `packages/deployment-readiness`.
- Read-only readiness service methods:
  - `getLLMIntegrationTestProfile()`
  - `listLLMIntegrationTestCases()`
  - `listLLMIntegrationTestSafetyChecks()`
  - `getLLMIntegrationTestReadinessSummary()`
  - `canRunLLMIntegrationLiveTests()`
- Read-only API endpoints:
  - `GET /readiness/llm-integration/profile`
  - `GET /readiness/llm-integration/test-cases`
  - `GET /readiness/llm-integration/safety-checks`
  - `GET /readiness/llm-integration/summary`
- Safe `/health` metadata under `llmIntegrationTests`.
- Dashboard read model and panel under `GET /dashboard/llm-integration`.
- Deterministic tests for readiness models, API routes, dashboard, health metadata, no-secret/no-env exposure, and a skipped live-test skeleton.

## What v1 Does Not Implement

- No live LLM call in default runtime or default tests.
- No BYOK, OAuth, device-code, WIF, IAM, or cloud identity exchange.
- No broad live multi-provider integration.
- No streaming or tool calling.
- No Local CLI or vendor CLI provider execution.
- No provider credential cache reads.
- No Vault/cloud secret backend integration.
- No MCP real transport.
- No deployment or production rollout.

## Required Environment Gates

Live tests are optional and skip unless all required gates are configured:

```text
AICHESTRA_LLM_INTEGRATION_TESTS=true
AICHESTRA_LLM_PROVIDER=openai_compatible
AICHESTRA_ENABLE_REMOTE_LLM=true
AICHESTRA_ALLOW_REMOTE_LLM_COMPLETION=true
AICHESTRA_LLM_BASE_URL=<configured but never returned>
AICHESTRA_LLM_API_KEY_SECRET_REF=<preferred SecretRef>
AICHESTRA_LLM_ALLOWED_MODELS=<explicit allowlist>
AICHESTRA_LLM_DEFAULT_MODEL=<member of allowlist>
AICHESTRA_LLM_ROUTING_MODE=single_provider
AICHESTRA_ENABLE_LLM_FALLBACK=false
AICHESTRA_LLM_MAX_FALLBACK_ATTEMPTS=0
AICHESTRA_LLM_TEST_BUDGET_USD=<small positive cap>
AICHESTRA_LLM_TEST_PROMPT_CLASS=integration_smoke
```

Controlled test-only `AICHESTRA_LLM_API_KEY` can satisfy the credential presence gate, but SecretRef remains preferred. If the env SecretRef provider is used, `AICHESTRA_ENABLE_ENV_SECRET_PROVIDER=true` and `AICHESTRA_ALLOWED_SECRET_ENV_KEYS` must include the referenced key.

## SecretRef Requirements

The profile prefers `AICHESTRA_LLM_API_KEY_SECRET_REF`. Readiness APIs only report whether a SecretRef is configured and whether the env SecretRef provider appears enabled; they do not resolve the secret, read the API key, return the SecretRef value, or enumerate environment values.

## Test Model Allowlist

Live tests require `AICHESTRA_LLM_ALLOWED_MODELS` and `AICHESTRA_LLM_DEFAULT_MODEL`. The default model must be in the allowlist. Dashboard and API surfaces return model counts and allowlist status only.

## Budget Cap Requirements

Live tests require `AICHESTRA_LLM_TEST_BUDGET_USD`. The v1 profile treats a missing cap as a skip condition until live tests are enabled, and treats invalid or excessive caps as unsafe. The profile maximum is intentionally small and meant for smoke tests only.

## Safe Test Prompt Rules

Live test prompts must be harmless, short, non-sensitive smoke prompts. They must not contain secrets, source credentials, customer data, credential cache contents, or proprietary payloads. Outputs must be preview-limited and redacted before usage, audit, dashboard, or health exposure.

## Live Test Skip Behavior

Missing gates skip optional live tests. Unsafe gates block live tests. Unsafe gates include:

- provider kind other than `openai_compatible`
- default model not in allowlist
- invalid or excessive budget cap
- fallback enabled or fallback attempts greater than zero
- streaming or tool-call flags enabled
- BYOK/OAuth/WIF/IAM flags enabled
- credential cache path references

## Usage Ledger Verification

The profile includes a gated live test case for verifying usage ledger metadata after a controlled remote completion. Default tests do not run that case. Usage metadata must not include API keys, env values, raw prompts, or raw provider responses.

## Audit and Redaction Verification

The profile includes audit/redaction checks for:

- API key not exposed
- raw env values not exposed
- raw prompts not stored
- raw provider responses not stored
- redacted output preview only
- no external audit export

## Fallback Restrictions

`AICHESTRA_ENABLE_LLM_FALLBACK=false` and `AICHESTRA_LLM_MAX_FALLBACK_ATTEMPTS=0` are required for this profile. Future bounded fallback integration tests must be separate and explicit.

## No Streaming or Tool Calls

Streaming and tool calling remain out of scope. Readiness flags fail unsafe configurations such as `AICHESTRA_ENABLE_LLM_STREAMING=true` or `AICHESTRA_ENABLE_LLM_TOOL_CALLS=true`.

## How To Run Safely

Use default validation first:

```text
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

Live LLM tests are optional, non-production only, and must be configured with all gates above. The v1 readiness API can be checked before enabling any live test:

```text
curl http://localhost:3000/readiness/llm-integration/summary
curl http://localhost:3000/dashboard/llm-integration
```

## Known Limitations

- This profile is readiness/test scaffolding, not production LLM integration.
- SecretRef values are metadata-only in readiness surfaces.
- No live provider is called by the profile service or dashboard.
- Staging Deployment Dry-run Profile v0 consumes this profile as an optional source and classifies it as skipped/gated/ready/blocked without executing live tests.
- Staging Release Candidate Checklist v0 consumes the skip classification and requires release notes to document the optional skip before RC designation.
- Staging Deployment Execution Plan v0 consumes the optional decision as a pre-deployment gate without running LLM tests.
- No streaming, tool calls, broad provider matrix, BYOK, OAuth, WIF/IAM, vendor CLI, credential cache, or production secret backend path exists.
