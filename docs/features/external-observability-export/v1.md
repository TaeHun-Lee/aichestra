# External Observability Export v1

Status: `v1_implemented_skeleton`.

External Observability Export v1 adds a disabled-by-default exporter boundary, safe export planning models, check-only envelope validation, and API/dashboard visibility. It does not send logs, metrics, traces, audit events, prompts, provider responses, or payloads to any external service.

## What v1 Implements

- `ObservabilityExporterConfig` provider metadata for disabled, mock, and future exporter options.
- `ObservabilityExportEnvelope` for safe audit, metric, trace, readiness, and future policy-shadow export planning.
- `ObservabilityExportSafetyCheck` and `ObservabilityExportReadinessSummary`.
- `ObservabilityExporter` interface with disabled, mock, and future-not-configured implementations.
- `ExternalObservabilityExportReadinessService` for config listing, safety checks, safe envelope building, envelope validation, summary metadata, and future backend listing.
- Read-only/check-only API routes:
  - `GET /observability/export/config`
  - `GET /observability/export/backends`
  - `GET /observability/export/safety-checks`
  - `GET /observability/export/summary`
  - `POST /observability/export/mock-envelope/check`
- `/dashboard/observability` External Observability Export panel metadata.
- `/health.observabilityExport` summary metadata.

## What v1 Does Not Implement

- Real OpenTelemetry export.
- External log, metric, trace, or audit delivery.
- Alert delivery.
- Durable export checkpoints.
- Retention deletion/export enforcement.
- Production observability backend.
- Production tenant-scoped export authorization.
- Datadog, Grafana Cloud, CloudWatch, OpenSearch, Elasticsearch, Splunk, SIEM, S3, GCS, Azure Monitor, Honeycomb, Prometheus Pushgateway, or custom backend calls.

## Exporter Options

`ObservabilityExporterConfig.exporterKind` supports:

- `disabled`
- `mock`
- `opentelemetry_future`
- `datadog_future`
- `grafana_cloud_future`
- `cloudwatch_future`
- `opensearch_future`
- `splunk_future`
- `siem_future`
- `s3_future`
- `custom_future`

Default runtime uses `disabled`. The mock exporter is for deterministic tests only. Future exporters return not-configured/future metadata and never call external backends.

## Export Envelope

`ObservabilityExportEnvelope` carries only bounded and sanitized metadata:

- `id`
- `envelopeKind`
- `source`
- optional `tenantId`, `teamId`, `projectId`, `resourceScope`
- `redactionClass`
- `retentionClass`
- `payloadSummary`
- `rawPayloadIncluded: false`
- `secretFieldsRedacted: true`
- sanitized `metadata`

The envelope never includes raw payloads, raw prompts, raw provider responses, credential material, tokens, cookies, session IDs, API keys, Vault secrets, GitHub tokens, webhook secrets, database URLs, or env values.

## Safety Checks

V1 checks:

- exporter disabled by default;
- no raw payload;
- no secret fields;
- no env values;
- redaction applied;
- tenant scope present or warning-only when missing;
- retention class present;
- endpoint values not exposed;
- auth values not exposed.

Failures block mock metadata recording and keep readiness blocked. Warnings are advisory metadata for future production design.

## Redaction, Retention, And Scope

Export planning consumes the existing Observability / Audit Retention v0 redaction and retention vocabulary. Future production export must carry a redaction class, retention class, tenant/team/project/resource scope, audit-query scope decision, Auth/RBAC context, Policy decision, and SecretRef-safe credential boundary before any external exporter can be considered.

## Guarantees

- External export runtime: disabled.
- External calls enabled: false.
- Raw payload export allowed: false.
- Secret export allowed: false.
- Endpoint values exposed: false.
- Auth values exposed: false.
- Env values exposed: false.

## Known Limitations

The service is in-memory and metadata-only. It has no durable queue, checkpoint, retry, dead-letter, backend credential store, alerting channel, retention deletion workflow, or production tenant-scoped authorization model.
