# Local Agent Protocol v1

## What v1 Adds Over v0

Local Agent Protocol v1 keeps the v0 mock-first boundary and adds a controlled local fixture daemon simulation for protocol lifecycle testing.

New v1 surfaces:

- mock signed channels and handshakes;
- fixture Local Agent daemon simulation;
- capability advertisements and negotiation;
- local CLI compatibility matrix and compatibility results;
- consent queue/history read models with `approved_once` and `approved_for_session`;
- invocation streams with deterministic redacted events;
- timeout, cancellation, disconnect, revoke, and session-expiry outcomes;
- API, health, dashboard, audit, policy, secrets/sandbox, and provider integration visibility.

v1 still does not implement a production Local Agent daemon, real WebSocket/gRPC/HTTP tunnel, PTY automation, vendor CLI execution, credential-cache access, provider calls, OAuth/device-code/WIF/IAM exchange, or secret forwarding.

## Trust Boundary

Aichestra Cloud owns metadata, policy decisions, consent records, audit records, and redacted output previews.

The future Aichestra Local Agent is a user-machine component. In v1 it is represented only by `FixtureLocalAgentDaemon`, `MockLocalAgentTransport`, and in-memory protocol repositories. The fixture daemon never opens sockets, spawns processes, executes vendor CLIs, reads credentials, or forwards secrets.

## Mock Signed Channel

`LocalAgentChannel` models a future transport channel:

- `mock_in_memory`
- `future_websocket`
- `future_grpc`
- `future_http_tunnel`

Only `mock_in_memory` is allowed in v1. Future channel kinds are blocked.

`LocalAgentHandshake` records a deterministic mock challenge/response. A successful mock response marks the channel `established` with `handshakeStatus = mock_verified`. This is not production cryptography and must not be treated as a secure channel implementation.

Expired, revoked, or disconnected channels cannot dispatch fixture invocations.

## Fixture Daemon Simulation

`FixtureLocalAgentDaemon` wraps `LocalAgentProtocolService` for deterministic fixture behavior:

- start and stop a fixture agent;
- connect or disconnect a mock channel;
- advertise capabilities;
- receive an invocation through the protocol service;
- expose deterministic emitted stream events;
- complete a fixture invocation.

It is in-memory only and does not use network, shell execution, PTY, vendor CLIs, or credential files.

## Capability Negotiation

`LocalAgentCapabilityAdvertisement` records advertised capabilities, provider templates, parser modes, consent levels, sandbox kinds, timeout limits, streaming support, and cancellation support.

Capabilities are not permissions. Dispatch still requires:

- compatible provider template and parser mode;
- policy allow/approval;
- required consent;
- active session;
- established mock channel for fixture agents;
- sandbox and redaction policy refs;
- no denied secret/network/danger capability.

Shell execution, network access, secret access, and danger/full-access remain disabled by default.

## CLI Compatibility Matrix

`LocalCliCompatibilityEntry` records fixture compatibility metadata for local CLI templates. v1 seeds:

- Claude headless JSON;
- Claude stream JSON;
- Codex exec;
- Codex JSONL;
- Gemini JSON;
- PTY fallback as unsupported/unstable.

`LocalCliCompatibilityResult` records deterministic fixture checks. v1 never detects real CLI versions and never executes `claude`, `codex`, `gemini`, or custom vendor commands.

Incompatible template, parser mode, version range, or unsupported PTY blocks invocation with a deterministic reason.

## Consent Workflow

Consent requests now include provider id, workspace ref, requested capability set, consent level, timeout, and safety notes.

Consent decisions support:

- `approved_once`
- `approved_for_session`
- `denied`
- `expired`

The legacy `approved` value remains accepted for v0 compatibility.

`approved_for_session` expires with the active `LocalAgentSession`. `danger_full_access` and `network_or_secret_access` remain denied regardless of user approval in v1. Shell execution remains denied unless a future explicit policy enables it. Read-only fixture flow can be approved.

## Streaming Lifecycle

`LocalAgentInvocationStream` tracks stream state and event count.

`LocalAgentStreamEvent` stores deterministic sequence, source, type, redacted preview payload, and timestamp. Stdout and stderr remain separate. Stderr progress does not automatically fail an invocation.

All stored payload previews are redacted and size-limited through Secrets/Sandbox redaction policy. Raw unredacted output is not stored.

## Timeout, Cancel, Disconnect, Revoke

v1 deterministic outcomes:

- disconnect before dispatch: `local_agent_unavailable`;
- disconnect during running: invocation becomes `failed` with `local_agent_disconnected`;
- timeout: invocation becomes `timed_out`;
- cancel before dispatch or during streaming: invocation becomes `cancelled`;
- completed invocation cannot be cancelled;
- revoked agents cannot emit events;
- expired sessions cannot dispatch.

Audit events are recorded for channel creation/establishment/revocation, fixture start/stop, capability advertisement, compatibility checks, stream events/completion, disconnect, timeout, cancellation, and redaction.

## Policy Integration

Policy-as-code v0 includes v1 Local Agent actions:

- `local_agent.channel.create`
- `local_agent.handshake.verify`
- `local_agent.capability.advertise`
- `local_agent.compatibility.check`
- `local_agent.stream.receive`
- `local_agent.cancel`
- `local_agent.session.approve`
- `local_agent.session.expire`
- `local_cli.template.use`

Defaults allow mock fixture channel and compatibility operations, deny real transport and PTY, deny danger/full-access, deny network/secret access, require consent for local CLI template use, and deny incompatible templates.

## Secrets and Sandbox Integration

Local Agent v1 requires sandbox and redaction references for local invocation. Network egress remains denied by default. Secret forwarding remains denied. No secret lease is issued by default.

Workspace refs, sandbox refs, network policy refs, redaction policy refs, and secret scope ids are metadata only. Output is redacted before protocol event, stream event, dashboard, API, and audit storage.

## Enterprise Provider Integration

`LocalCliProviderAdapter` routes through `LocalAgentProtocolService`.

Expected v1 outcomes:

- no agent: `local_agent_required`;
- disconnected agent: `local_agent_unavailable`;
- connected fixture agent without channel: `channel_required`;
- missing consent: `awaiting_consent`;
- denied consent: `consent_denied`;
- incompatible template: `provider_template_incompatible`;
- compatible fixture agent plus approved read-only consent: deterministic `mock_completed` fixture result.

Cloud-side direct local CLI execution remains blocked and audited.

## Dashboard Changes

The dashboard shows fixture agents, mock channel and handshake status, capability advertisements, compatibility entries/results, consent queue/history, invocation stream events, redaction status, timeout/cancel/disconnect examples, and blocked real transport/vendor CLI/PTY examples.

It is not a desktop Local Agent UI.

## Safety Constraints

- No real daemon.
- No real transport.
- No production crypto.
- No vendor CLI execution.
- No PTY automation.
- No credential cache reads/uploads.
- No real LLM/provider calls.
- No remote Git operations.
- No secret forwarding or injection.
- No danger/full-access mode.

## Test Strategy

`tests/local-agent-protocol-v1.test.ts` covers mock channels, handshake success/failure, expired/revoked channel blocks, fixture daemon lifecycle, capability advertisement, compatibility matrix behavior, consent queue/history, streaming event ordering/redaction, timeout/cancel/disconnect/revoke outcomes, enterprise provider v1 states, policy decisions, API endpoints, health, and no-secret exposure.

Existing v0, provider, security, policy, runner, dashboard, and full regression tests remain in the validation set.

## Known Limitations

- Runtime repositories are in-memory only.
- Postgres tables are schema skeleton only.
- Mock signatures are deterministic metadata only.
- Compatibility uses fixture versions only.
- No real Local Agent pairing or transport exists.
- No production consent UX exists.
- No real vendor CLI execution exists.
- No production sandbox or network enforcement exists.
