# Merge Queue Live Integration-Test Profile v1

Status: v1_implemented

Merge Queue Live Integration-Test Profile v1 defines a controlled, skipped-by-default profile for validating merge queue readiness against non-production Git fixtures. It is read-only and safe by default. Default runtime and tests never execute a real merge, run remote Git, force-push, delete branches, or update remote PRs.

## What v1 Implements

- Models:
  - `MergeQueueIntegrationTestProfile`
  - `MergeQueueIntegrationTestCase`
  - `MergeQueueIntegrationSafetyCheck`
  - `MergeQueueIntegrationTestReadinessSummary`
- Readiness service methods:
  - `getMergeQueueIntegrationTestProfile()`
  - `listMergeQueueIntegrationTestCases()`
  - `listMergeQueueIntegrationSafetyChecks({ category? })`
  - `canRunMergeQueueIntegrationLiveTests()`
  - `getMergeQueueIntegrationTestReadinessSummary()`
- Read-only API endpoints:
  - `GET /readiness/merge-queue-integration/profile`
  - `GET /readiness/merge-queue-integration/test-cases`
  - `GET /readiness/merge-queue-integration/safety-checks`
  - `GET /readiness/merge-queue-integration/summary`
- Health metadata key `mergeQueueIntegrationTests`.
- Dashboard read model and panel: `GET /dashboard/merge-queue-integration` and `Merge Queue Live Integration Tests` section in the rendered HTML.
- A skipped-by-default live test skeleton (`tests/merge-queue-live-integration-test-profile-v1.test.ts`).

## What v1 Does Not Implement

- Real merge execution.
- Auto-merge.
- Remote merge, rebase, force-push, or branch-delete calls.
- Live remote Git fetch/push/checkout/switch in default runtime/tests.
- Vendor CLI execution.
- LLM calls.
- Workspace mutation.
- GitLab or Bitbucket.
- Production rollout.

## Required Gates

Live tests can run only when every required gate is configured and no unsafe gate is present:

| Gate | Required value |
|---|---|
| `AICHESTRA_MERGE_QUEUE_INTEGRATION_TESTS` | `true` |
| `AICHESTRA_ENABLE_REMOTE_GIT` | `true` |
| `AICHESTRA_GITHUB_INTEGRATION_TESTS` or `AICHESTRA_GITHUB_APP_INTEGRATION_TESTS` | `true` |
| `AICHESTRA_GIT_PROVIDER` | `github` |
| `AICHESTRA_GIT_ALLOWED_REPOS` | non-empty allowlist |
| `AICHESTRA_GIT_ALLOWED_BRANCH_PREFIX` | `aichestra/test/` |
| `AICHESTRA_ALLOW_REMOTE_MERGE` | `false` |
| `AICHESTRA_ALLOW_REMOTE_REBASE` | `false` |
| `AICHESTRA_ALLOW_REMOTE_FORCE_PUSH` | `false` |
| `AICHESTRA_ALLOW_REMOTE_BRANCH_DELETE` | `false` |
| `AICHESTRA_MERGE_QUEUE_DRY_RUN_ONLY` | `true` |
| `AICHESTRA_TEST_MERGE_QUEUE_REPO` | allowlisted non-production repo |
| `AICHESTRA_TEST_MERGE_QUEUE_BASE_BRANCH` | safe non-production base branch |
| `AICHESTRA_TEST_MERGE_QUEUE_SOURCE_BRANCHES` | csv of branches under `aichestra/test/` |

## Skip Behavior

Missing gates skip optional live tests; the default test suite still passes. Unsafe gates make `canRunLiveTests=false` and block live readiness.

Unsafe states reported as unsafe gates:

- `AICHESTRA_ALLOW_REMOTE_MERGE=true`
- `AICHESTRA_ALLOW_REMOTE_REBASE=true`
- `AICHESTRA_ALLOW_REMOTE_FORCE_PUSH=true`
- `AICHESTRA_ALLOW_REMOTE_BRANCH_DELETE=true`
- `AICHESTRA_MERGE_QUEUE_DRY_RUN_ONLY=false`
- `AICHESTRA_ENABLE_AUTO_MERGE=true`
- `AICHESTRA_GIT_PROVIDER` other than `github`
- Branch prefix other than `aichestra/test/`
- Configured test repo not present in the allowlist
- Test source branches outside the configured prefix
- Test base branch overlapping test source branches

## Test Repository and Branch Requirements

The test repo must be allowlisted via `AICHESTRA_GIT_ALLOWED_REPOS` and pointed to a non-production fixture repository owned for integration testing. Test branches must use the `aichestra/test/` prefix. The base branch must differ from the source branches. The readiness API/dashboard return counts and booleans only and never return env values, repo URLs, or branch names.

## Dry-Run-Only Live Validation

Live validation, when gates allow, is limited to safe metadata and the existing local/mock dry-run path:

- merge queue policy evaluation against test branch metadata,
- local dry-run merge simulation,
- branch lease metadata check,
- conflict risk metadata check,
- policy decision metadata check,
- cleanup metadata check.

No real merge, fetch, push, rebase, force-push, branch deletion, or remote PR update is performed by this profile.

## No-Auto-Merge Guarantee

The profile records `noAutoMerge: true`, `noForcePush: true`, `noBranchDelete: true`, and `realMergeExecuted: false`. The safety checks `no_auto_merge`, `no_force_push`, and `no_branch_delete` fail if any unsafe gate is set.

## Cleanup Policy

Cleanup remains manual mark-only in v1. Branch deletion is forbidden, including for cleanup. The cleanup safety check confirms `manualMarkOnly: true` and `branchDeletionAllowed: false`.

## API and Dashboard Surfaces

`/readiness/merge-queue-integration/*` endpoints are read-only planning models. `/dashboard/merge-queue-integration` returns a `MergeQueueIntegrationTestReadModel` with summary, profile, test cases, gated live cases, mock cases, safety checks, gate status, operation policy, cleanup policy, and no-secret status. Dashboard sanitization replaces secret-adjacent values and credential cache paths with redacted placeholders. The rendered dashboard HTML adds a `Merge Queue Live Integration Tests` panel.

## Audit and Observability Expectations

Audit events for this profile record skip reasons when gates are missing, sanitized safety-check outcomes, and dashboard read-only events. They never include env values, repo URLs, branch names from env, or remote merge calls (none are made). No external observability export is added.

## Skipped-by-default Behavior

The live test skeleton (`tests/merge-queue-live-integration-test-profile-v1.test.ts`) is skipped by default because v1 does not implement a live merge queue execution path. Default tests run only the deterministic mock-path checks.

## Known Limitations

- Live test skeleton is intentionally skipped even when all gates are configured because v1 does not implement live merge queue execution.
- Cleanup automation is not implemented.
- Production rollout remains blocked by production auth, real secret backend, DB operations, and production webhook hardening.
- Repo allowlist scope is enforced by metadata only; production tenant enforcement and scope filtering remain future work.
- Real Merge Execution Policy v1 is now the required disabled policy/readiness boundary before any future live merge execution task. It does not change this profile's skipped-by-default dry-run/readiness behavior.
