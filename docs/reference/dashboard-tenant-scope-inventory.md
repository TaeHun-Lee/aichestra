# Dashboard Tenant Scope Inventory

Status:

- Dashboard/Readiness Tenant Scope Planning: `v1_implemented`
- Dashboard/Readiness Tenant Scope Implementation: `v1_implemented`
- Tenant Scope Enforcement: `v1_implemented_partial`
- Dashboard Scope Filtering: `v1_implemented_partial`

This inventory is planning/readiness metadata only. It does not implement production dashboard filtering, production tenant isolation, row-level security, tenant provisioning, or production Auth/RBAC.

Implementation v1 exposes the inventory through `DashboardPanelScopeSummary` and optional dashboard section `ScopedReadModelMetadata`. These records add safe required-scope, available-scope, missing-scope, role visibility, redaction, and fallback hints while keeping `tenantFilteringImplemented: false` and `productionEnforcementImplemented: false`.

Tenant Scope Enforcement v1 adds representative `enforcementMode` and `scopeDecisionSummary` metadata to dashboard panels. It is partial metadata scaffolding only: panels are not hidden, dashboard records are not tenant-filtered, and production tenant enforcement remains false.

Dashboard Scope Filtering v1 (`docs/features/dashboard-scope-filtering/v1.md`) consumes this inventory to produce `DashboardPanelFilterDecision` rows when a non-default role/scope context is supplied via safe demo headers. It redacts representative sensitive panels (`security`, `vaultSecretBackend`, `observability`, `audit`, `githubApp`, `githubAppIntegration`, `llmIntegration`, `mcp`, `staging`, `stagingDryRun`, `stagingReleaseCandidate`, `stagingExecution`) for non-admin roles and exposes the decision summary through `/dashboard/scope-filter`. `productionFiltering`, `productionAuthImplemented`, `productionTenantEnforcement`, and `externalIdentityProviderCalled` all remain `false`.

## Summary

- Current dashboard visibility is read-only metadata with no production tenant filtering.
- Target production behavior requires tenant-scoped dashboard read models and role-aware redaction.
- Secret-adjacent panels must remain value-redacted regardless of role.
- Audit and observability panels must require explicit audit query scope before production.
- Dashboard/Readiness Tenant Scope Planning v1 adds inventory and planning surfaces only.
- Dashboard/Readiness Tenant Scope Implementation v1 adds safe scope summaries, missing-scope warnings, role visibility hints, and redaction labels only.
- Tenant Scope Enforcement v1 adds partial representative enforcement summaries only; it does not implement production dashboard filtering.
- Registry Tenant Scope Enforcement v1 adds registry-panel scope summaries only; it does not implement production registry filtering or tenant isolation.
- Registry Artifact Trust v1 adds registry-panel digest/mock-signature/provenance summaries only; it does not implement real signing, real verification, or production artifact trust enforcement.
