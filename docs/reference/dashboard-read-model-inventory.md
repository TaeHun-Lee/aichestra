## Notes

- API-backed dashboard endpoints must not execute provider adapters, GitHub calls, LLM calls, runner commands, MCP tools, Local Agent fixture invocations, live readiness checks, dry-run integration tests, deployment actions, or workflow runs.
- GitHub App / Webhook Hardening read models are planning/runtime-status only and must not create apps, mint live installation tokens, read private keys, enable production webhooks, call GitHub, or expose secrets. GitHub App Controlled Implementation v1 may show mock token-handle readiness metadata only.
- GitHub App integration-test read models are skipped-by-default readiness metadata only and must not call GitHub, mint live installation tokens, read private keys, run branch/PR tests, auto-merge, force-push, delete branches, or expose env values/secrets.
- Vault integration-test read models are skipped-by-default readiness metadata only and must not call Vault in dashboard/API/default tests, expose Vault token/address/path/key/secret/env values, write/delete/rotate secrets, list broad paths, or read credential caches.
- Demo fallback is intentionally allowed for offline/static tests, but it must remain explicit and documented.
- All read models are sanitized before API/web exposure.
- Auth/RBAC read models expose mock identity metadata only and do not expose sessions, tokens, cookies, passwords, or provider credentials.
- Auth/RBAC Production Readiness read models are planning-only and do not call IdPs, issue sessions/JWTs, store cookies, issue service-account credentials, or expose tokens, cookies, session ids, passwords, raw identity assertions, IdP client secrets, or provider credentials.
- Observability read models are read-only and do not call external observability services, export audit logs, deliver alerts, run retention deletion jobs, or expose raw secrets/tokens/payloads/prompts/outputs.
- Database operations read models are read-only and do not connect to production databases, execute migrations, run backup/restore jobs, delete data, run partition maintenance, or expose `DATABASE_URL`, `AICHESTRA_DATABASE_URL`, or `AICHESTRA_TEST_DATABASE_URL` values.
- Production secret backend decision read models are read-only and do not call Vault/cloud/custom secret backends, read/migrate/rotate secrets, issue production credentials, read credential caches, expose env/secret values, or mark a production backend implemented.
- Vault secret backend read models are read-only and do not call Vault in default runtime/tests, expose Vault tokens or secret values, list broad Vault paths, migrate/rotate secrets, issue production credentials, read credential caches, expose env values, or mark secrets production-ready.
- Staging RC read models are read-only and do not create releases, Git tags, GitHub releases, deploy, run CI jobs, execute remote integration tests, call providers, mutate resources, or expose env/secret values.
- Staging deployment execution read models are read-only and do not create releases, Git tags, deploy, run deployment commands, run CI jobs, execute remote integration tests, call providers, mutate resources, issue credentials, or expose env/secret values.
- Staging CI/CD read models are read-only and do not create active workflows, run CI jobs, deploy, call external providers, run remote integration tests by default, upload artifacts, or expose env/secret values.
