# Policy Domain Mapping

Status: v0 planning inventory

This document maps current static TypeScript policy domains to future policy bundle representations. It does not enable OPA, Cedar, dynamic policy execution, remote bundle loading, or production policy changes.

| Domain | Current actions | Resource kinds | Static location | Required AuthContext fields | Required resource context fields | Required gates | Current audit | Future bundle | Gaps |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Git branch/PR/webhook/merge/rebase | `git.branch.create`, `git.pull_request.create`, `git.remote_operation`, `git.webhook.process`, merge/rebase denials | repo, branch, pull request, webhook delivery | `packages/policy` | actor, principal, auth mode, roles | repo id, branch, allowed repo, delivery id | remote Git env gates, repo allowlist, branch prefix, signature verification | policy audit, git audit, webhook audit | `git-remote-operations`, `git-webhook-receiver` | GitHub App actor and tenant scope future |
| LLM routing/completion/fallback/model use | `llm.completion`, `llm.remote_completion`, `llm.model.use` | provider, model, virtual key, task | `packages/policy` | actor, principal, roles | provider id, model id, budget, task id, run id | remote LLM gates, model allowlist, budget, SecretRef | policy audit, LLM audit, usage ledger | `llm-provider-routing` | tenant budgets future |
| MCP server/tool invocation | `mcp.server.list`, `mcp.tool.invoke`, high-risk denials | MCP server, tool | `packages/policy` | actor, principal, roles | tool id, risk, input redaction, server id | mock gateway, risk allowlist, network deny | policy audit, MCP audit | `mcp-tool-risk` | real transport governance future |
| Runner execution/command/workspace | `runner.execute`, `runner.command.execute`, `runner.secret.inject` | runner, command, workspace | `packages/policy` | actor, principal, roles | command, args, workspace, harness policy | command disabled by default, fixture workspace, network deny | policy audit, runner audit | `runner-execution` | production sandbox future |
| Registry mutation/approval/rollback | `registry.create`, `registry.update`, `registry.approve`, `registry.rollback` | skill, harness, instruction, package | `packages/policy` and registry authorizer | actor, principal, roles | lifecycle, approval, eval, checksum | approval/eval/checksum gates | policy audit, registry audit | `registry-governance` | production reviewer identity future |
| Improvement proposal/apply gate | `improvement.proposal.approve`, `improvement.apply` | proposal, draft change, eval, canary | `packages/policy` and governance gates | actor, principal, roles | eval result, canary status, approval state | auto apply false, human approval, eval, canary | policy audit, governance audit | `improvement-governance` | real eval/canary future |
| SecretRef credential resolution | `provider.credential.resolve`, `secret.read`, `secret.lease.request` | SecretRef, lease, provider credential | `packages/policy` | actor, principal, roles, purpose | secret ref id, secret kind, provider id, lease | SecretRef active, purpose allowed, env allowlist | policy audit, credential audit, security audit | `secretref-resolution` | real backend and tenant scope future |
| Secrets/Sandbox/network egress | `network.egress`, `runner.secret.inject`, sandbox decisions | sandbox profile, network policy, secret scope | `packages/policy` | actor, principal, roles | sandbox profile, destination, command, scope | network default deny, no production injection | policy audit, security audit | `sandbox-network` | real sandbox runtime future |
| Enterprise Provider/local CLI/PTY/cache | `provider.invoke`, `provider.credential.resolve`, local CLI/PTY/cache denials | provider, adapter, local CLI session | `packages/policy` | actor, principal, roles | provider id, auth type, credential access | local CLI disabled, cache never read | policy audit, provider audit | `enterprise-provider-boundary` | service account identity future |
| Local Agent invoke/consent/capability/stream | `local_agent.invoke`, `local_agent.consent.approve`, credential-cache denial | local agent, host, consent, stream | `packages/policy` | actor, principal, roles | agent id, host id, consent id, capability | mock transport, fixture daemon, consent | policy audit, local agent audit | `local-agent-consent` | real daemon/device trust future |
| Auth/RBAC authorization | `policy.evaluate`, authorization bridge | principal, actor, role, scope | `packages/auth`, `packages/policy` | principal, actor, roles, teams | scope, resource, action | mock auth only | auth audit, policy audit | `auth-subject-mapping` | production IdP, tenant enforcement, sessions future |
| Dashboard/read-model access | `dashboard.read`, audit/readiness reads | dashboard panel, audit/read model | API/dashboard read model | actor, principal, roles | panel, retention class, tenant | read-only routes, sanitizer | auth audit, dashboard/readiness metadata | `dashboard-read-scope` | tenant-scoped filtering future |
| Deployment readiness | `readiness.read` | profile, check, risk | `packages/deployment-readiness` | actor, principal, roles | profile, category | read-only deterministic data | readiness metadata | `readiness-access` | production profile enforcement future |

All future bundles must preserve deny-by-default behavior and must not bypass Auth/RBAC, SecretRef, Git, LLM, MCP, Runner, Local Agent, Dashboard, Observability, or Secrets/Sandbox gates.
