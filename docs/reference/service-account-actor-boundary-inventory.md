## Notes

- Active entries are mock/local metadata only.
- No service-account credentials, JWTs, API keys, private keys, client secrets, or installation tokens are issued.
- API-request paths should keep using request-derived `RequestContext`/`AuthContext`; service-account fallback is for explicit system/service work.
- Service-account contexts can carry optional tenant/team/project/resource scopes when callers supply mock/readiness metadata; this does not enforce production tenancy.
- Policy denials remain authoritative.
- Production tenant/team/project/repo enforcement remains future work.
